Was ist ein Canonical-Tag?

🔗

Was er tut

Der rel="canonical" Tag teilt Google mit, welche URL die "offizielle" Version einer Seite ist. Bei Duplicate Content verhindert er, dass Google die falsche Seite indexiert.

📋

Korrektes Format

Muss im <head> stehen, absolute URL verwenden (mit https://), nur einmal pro Seite vorhanden sein.

<link rel="canonical" href="https://example.de/seite/">
⚠️

Typische Fehler

  • • Kein Canonical-Tag vorhanden
  • • Mehrere Canonical-Tags pro Seite
  • • Relative statt absolute URL
  • • Canonical zeigt auf andere Seite (Cross-Canonical)
  • • HTTP statt HTTPS im Canonical
✅

Best Practices

  • • Immer HTTPS verwenden
  • • Trailing Slash konsistent setzen
  • • Auf jeder Seite einen Self-Canonical setzen
  • • Bei Duplikaten auf die kanonische Version verweisen
