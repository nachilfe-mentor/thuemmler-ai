⬜ LCP — Ladezeit

Misst, wann das größte sichtbare Element (Hero-Bild, H1, Textblock) vollständig geladen ist. Wichtigster Wert für die gefühlte Ladegeschwindigkeit.

Gut: ≤ 2,5 Sekunden
Verbesserung nötig: 2,5 – 4,0 s
Schlecht: > 4,0 Sekunden

Häufige Ursachen

  • Großes, unkomprimiertes Hero-Bild
  • Langsamer Server (hoher TTFB)
  • Render-Blocking-CSS oder -JS
  • Kein CDN / weiter Server-Standort
  • Fehlender Lazy-Loading-Ausschluss für Hero

⬜ CLS — Layoutstabilität

Misst, wie stark sich das Layout während des Ladens verschiebt. Ein CLS von 0,1 bedeutet 10% der Seitenfläche verschiebt sich — das nervt Nutzer und kostet Klicks.

Gut: ≤ 0,1
Verbesserung nötig: 0,1 – 0,25
Schlecht: > 0,25

Häufige Ursachen

  • Bilder ohne width/height-Attribute
  • Ads und Embeds ohne feste Dimensionen
  • Web Fonts mit FOUT (Flash of Unstyled Text)
  • Dynamisch eingefügte Inhalte über bestehenden Elementen
  • Animationen die Layout-Properties ändern

⬜ INP — Interaktivität

Misst die Zeit vom Klick/Tippen bis zur nächsten visuellen Reaktion der Seite. Seit März 2024 ersetzt INP den alten FID-Wert als Kernmetrik.

Gut: ≤ 200 ms
Verbesserung nötig: 200 – 500 ms
Schlecht: > 500 ms

Häufige Ursachen

  • Schwerer JavaScript-Hauptthread blockiert
  • Zu viele Event-Listener auf einem Element
  • Synchrone Third-Party-Scripts (Analytics, Chat)
  • Große JavaScript-Bundles ohne Code Splitting
  • DOM-Manipulationen die Layout-Reflow auslösen

Schnellreferenz: Alle Grenzwerte auf einen Blick

Metrik Gut ✓ Verbesserung nötig ⚠ Schlecht ✗
LCP (Ladezeit) ≤ 2,5 s 2,5 – 4,0 s > 4,0 s
CLS (Layoutstabilität) ≤ 0,1 0,1 – 0,25 > 0,25
INP (Interaktivität) ≤ 200 ms 200 – 500 ms > 500 ms
